###### 反向面试运维篇
###### 职责
```
1、On-call (电话值班)的计划或者规定是什么？值班或者遇到问题加班时候有加班费吗？响应时间是多久，有A/B角色划分法嘛
2、我的日常工作是什么？
工作主线，负责业务(维护的业务线)
是否有入职培训，主要针对那方面的
每天预期/核心工作时间是多少小时？
在你看来，这个工作做到什么程度算成功？
如果推动运维相关的工作难度在那(开发、架构对自动化是否体会，现在自动化进度)
我入职的岗位是新增还是接替之前离职的同事？(是否有技术债需要还)？(zh)
入职之后在哪个项目组，项目是新成立还是已有的？(zh)负责那个业务线，现在盈利规模，核心业务等等
```
###### 技术
```
1、公司现在的整体架构是什么样的
2、技术栈是什么样的
3、容器化落地了嘛，后面有没有容器落地的方案
4、你们怎么集成和部署代码改动？是使用持续集成和持续部署吗？
5、灰度发布/金丝雀发布，现在的实现方案
6、有没有容灾的方案
7、是否有故障预演方案(具体是怎么实现的)
8、公司是否有技术分享交流活动？有的话，多久一次呢？(zh)
9、业务需求有没有文档记录？是如何记录的？(zh)运维部署实施方案和规范
```
###### 团队
```
1、团队内交流是什么样的
2、如果有不同的方案如何进行处理
3、生产发生事故。(是先进行处罚还是定位问题，责任等等)
4、生产发布现在的规范流程是什么样的
5、如何评定业务的优先级
```
###### 公司
```
1、有没有会议/旅行预算？使用的规定是什么？
2、晋升流程是怎样的？要求/预期是怎样沟通的？
3、工资涨幅是怎么算的，薪资构成是有那些构成，补助等等，年终福利
4、有公司级别的学习资源吗？比如电子书订阅或者在线课程？
5、我可以为开源项目做贡献吗？是否需要审批？
```
